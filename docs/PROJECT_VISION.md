# Project Vision

This repository is an AI-first development foundation for future 3D games built with Godot 4, typed GDScript, Synty-style assets, Blender-assisted asset processing, editor tools, and modular reusable systems.

It is not a game yet. It deliberately avoids theme, story, genre, and gameplay decisions so the foundation can support many future projects.

## Why This Exists

The goal is to make future game creation faster, cleaner, and more enjoyable by investing early in reusable infrastructure:

- Clear project structure.
- Small, understandable modules.
- Strong documentation.
- Repeatable asset pipelines.
- Godot editor tooling.
- Automated validation.
- AI-friendly conventions.

Instead of building isolated prototypes that become difficult to maintain, this repository should grow into a professional framework for rapid 3D game development.

## AI Collaboration Model

AI agents should treat this repository as a long-lived engineering system. Every change should make the next change easier to understand, test, document, or automate.

Agents should:

- Read the documentation before changing architecture.
- Keep work small and well-scoped.
- Preserve generic reusable naming.
- Explain tradeoffs when proposing new systems.
- Avoid creative decisions unless explicitly requested.
- Update docs alongside structural or architectural changes.

The human user remains the Creative Director and Technical Lead. AI may recommend engineering paths, but creative direction and major architecture approvals belong to the human lead.

## Success Criteria

This foundation succeeds when future projects can quickly assemble game-specific prototypes from reusable systems without rewriting import tools, editor workflows, validation logic, or core architecture.
