# Roadmap

This roadmap is staged to keep the project generic and reusable before any game direction is selected.

## Phase 1: Project Foundation

- Establish repository structure.
- Create core documentation.
- Define AI agent guidance.
- Add initial Godot project shell.
- Keep all folders theme-neutral.

## Phase 2: Asset Pipeline

- Design Blender-assisted Synty asset processing.
- Define source asset and processed asset conventions.
- Add transform normalization scripts.
- Plan collision mesh generation.
- Generate import reports.
- Add asset validation checks.

## Phase 3: Core Systems

- Add approved reusable runtime systems.
- Start with systems that unlock broad prototyping value.
- Keep all systems data-driven and genre-neutral.
- Document each system contract.
- Add tests for reusable logic.

## Phase 4: Editor Tools

- Build Godot editor plugins for repeatable workflows.
- Prioritize validators and import helpers before creative generation tools.
- Add tools such as prop scatter, road generation, forest generation, interior population, and navigation helpers only when requirements are clear.

## Phase 5: Gameplay Framework

- Create reusable gameplay building blocks after approval.
- Keep framework code separate from project-specific content.
- Add sample scenes only when needed to validate reusable systems.

## Phase 6: First Vertical Slice

- Select a small approved use case to validate the framework.
- Exercise asset import, runtime systems, editor tooling, and validation.
- Identify framework gaps.

## Phase 7: Theme Selection

- Choose a concrete game direction only after the foundation can support rapid iteration.
- Move game-specific content into clearly separated folders.
- Preserve reusable systems as framework-level assets.

## Current Status

Phase 1 is in progress.
